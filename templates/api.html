Методы API

'/api/students' - ['GET']

Данный метод возвращает информацию об учениках и набранных ими баллах в виде JSON-объекта. Не требует передачи параметров.

Пример возвращаемого результата:

{
    "students": [
        {
            "ID": 1,
            "Name": "Bob",
            "Math": 80,
            "Rus": 50
        },
        {
            "ID": 2,
            "Name": "Mark",
            "Math": 60,
            "Rus": 70
        },
        {
            "ID": 3,
            "Name": "Anna",
            "Math": 80,
            "Rus": 60
        }
    ]
}

Для каждого ученика определен следующий набор атрибутов:

  • 'ID' – идентификатор ученика;
  • 'Name' – имя ученика;
  • 'Math' – баллы ученика по математике;
  • 'Rus' – баллы ученика по русскому языку.

'/api/students/<int:ID>' - ['GET']

Данный метод возвращает по ID информацию об ученике и набранных им баллах в виде JSON-объекта. Передается в URL один параметр ID – идентификатор ученика. Параметр ID целого типа.

Пример возвращаемого JSON-объекта:

{
    "student": {
        "ID": 1,
        "Name": "Bob",
        "Math": 80,
        "Rus": 50
    }
}

Для каждого ученика определен следующий набор атрибутов:

  • 'ID' – идентификатор ученика;
  • 'Name' – имя ученика;
  • 'Math' – баллы ученика по математике;
  • 'Rus' – баллы ученика по русскому языку.

Метод может вернуть в случае если пользователя с таким ID не существует следующий JSON-объект:

{
    "error": "Not found"
}

'/api/users' - ['POST']

Данный метод передает информацию о выполнении задания. В запросе необходимо отправить JSON-объект с вашей фамилией, именем и ответами на задания.

Пример отправляемого JSON-объекта:

{
    "Name": "Ivan",
    "Surname": "Petrov",
    "Key1": 100,
    "Key2": 100,
    "Key3": 100
}

Значения атрибутов:

  • 'Name' – Ваше имя;
  • 'Surname' – Ваша фамилия;
  • 'Key1' – Баллы по математике ученика с ID = 7;
  • 'Key2' – Наибольшая сумма баллов по русскому языку и математике;
  • 'Key3' – Средний балл по русскому языку (только целая часть числа);

Атрибуты 'Key1', 'Key2', 'Key3' не являются обязательными в запросе, но правильные отправленные их значения, добавляют вам дополнительные баллы.

При правильном выполнении запроса вы получите JSON-объект:

{
    "success": "OK",
    "result": 3
}

В нем атрибут 'result' показывает сколько из атрибутов 'Key1', 'Key2', 'Key3' заполнено верными данными.

Возможен возврат JSON-объекта со следующие ошибками:

  • не передан в запрос JSON-объект: 
    {
    "error": "Empty request"
}
  • не переданы все обязательные параметры: 
    {
    "error": "Bad request"
}
  • неверный тип атрибута с ключом 'Name': 
    {
    "error": "Name not str"
}
  • неверный тип атрибута с ключом 'Surname': 
    {
    "error": "Surname not str"
}
  • неверный тип атрибута с ключом 'Key1': 
    {
    "error": "Key1 not int"
}
  • неверный тип атрибута с ключом 'Key2': 
    {
    "error": "Key2 not int"
}
  • неверный тип атрибута с ключом 'Key3': 
    {
    "error": "Key3 not int"
}
